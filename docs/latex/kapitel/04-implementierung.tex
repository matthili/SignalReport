\section{Implementierung}
\label{sec:implementierung}

\subsection{Mess-Engine}
Die Mess-Engine implementiert drei Messarten mit dynamischen Zielen:

\begin{itemize}
    \item \textbf{Ping-Messung}: Nutzt \texttt{InetAddress.isReachable()} mit konfigurierbarem Ziel
    \item \textbf{DNS-Messung}: Nutzt \texttt{InetAddress.getByName()} mit konfigurierbarem Hostname
    \item \textbf{HTTP-Messung}: Nutzt \texttt{java.net.http.HttpClient} mit konfigurierbarer URL
\end{itemize}

\begin{lstlisting}[language=Java, caption=Beispiel für Ping-Messung mit dynamischem Ziel]
public Measurement measure(String target) throws Exception {
    long start = System.nanoTime();
    boolean reachable = InetAddress.getByName(target).isReachable(3000);
    long end = System.nanoTime();
    double latency = (end - start) / 1_000_000.0;
    return new Measurement(target, latency, reachable, "PING");
}
\end{lstlisting}

\subsection{IP-Tracking und Host-Identifikation}
Jede Messung speichert zusätzlich Netzwerk-Informationen:

\begin{itemize}
    \item Lokale IPv4- und IPv6-Adressen
    \item Externe IPv4- und IPv6-Adressen (über ipify.org)
    \item Eindeutiger Host-Hash (basierend auf Hostname, OS und MAC-Adresse)
\end{itemize}

Die IP-Tracking-Logik erkennt automatisch Änderungen der externen IP:

\begin{lstlisting}[language=Java, caption=IP-Änderungserkennung]
public void trackIpChange(String currentIp, String hostHash) throws SQLException {
    String lastIp = getLastKnownIp(hostHash);
    if (lastIp == null) {
        recordIpChange(null, currentIp, "INITIAL", hostHash);
    } else if (!lastIp.equals(currentIp)) {
        recordIpChange(lastIp, currentIp, "CHANGE", hostHash);
    }
}
\end{lstlisting}

\subsection{Statistische Auswertung}
Die statistische Auswertung berechnet folgende Kennzahlen:

\begin{itemize}
    \item \textbf{Durchschnittliche Latenz}: $\mu = \frac{1}{n}\sum_{i=1}^{n}x_i$
    \item \textbf{95th Percentile}: Wert, unter dem 95\% der Messungen liegen
    \item \textbf{Paketverlust}: $\frac{\text{fehlgeschlagene Messungen}}{\text{gesamte Messungen}} \times 100$
    \item \textbf{Jitter}: Durchschnitt der Differenzen zwischen aufeinanderfolgenden Messungen
\end{itemize}

\begin{lstlisting}[language=Java, caption=Beispiel für 95th Percentile-Berechnung]
// Statistiken aus der Datenbank holen
List<Double> latencies = new ArrayList<>();
// ... Daten laden ...
Collections.sort(latencies);
int p95Index = (int) Math.ceil(latencies.size() * 0.95) - 1;
double p95Latency = latencies.get(p95Index);
\end{lstlisting}

\subsection{Web-Interface}
Das Web-Interface wurde mit Javalin implementiert und bietet:

\begin{itemize}
    \item \textbf{REST-API}: Für Datenzugriff (z. B. \texttt{/api/measurements})
    \item \textbf{Live-Chart}: Aktualisiert sich alle 5 Sekunden mit Chart.js
    \item \textbf{Heatmap}: Zeigt Latenzentwicklung pro Stunde
    \item \textbf{Statistik-Panel}: Sofortige Einschätzung der Netzwerkqualität
    \item \textbf{Konfigurations-UI}: Änderung von Messzielen und Einstellungen
    \item \textbf{Setup-Wizard}: Erstmalige Einrichtung über Web-Oberfläche
\end{itemize}

\subsection{PDF-Export}
Der PDF-Export mit OpenPDF generiert professionelle Berichte mit:

\begin{itemize}
    \item Kundendaten (Name, Provider, Kundennummer)
    \item Host-Informationen (Hostname, Hash, IPs)
    \item 3 Charts (PING/DNS/HTTP) mit roten Linien bei Ziel-Änderungen
    \item Ziel-Informationen unter jedem Chart
    \item Top 10 schlechteste Messungen
    \item Top 10 längste Verbindungsausfälle
    \item Gesamtanzahl der Ausfälle
\end{itemize}

\begin{lstlisting}[language=Java, caption=PDF-Chart-Generierung]
BufferedImage pingChart = createLatencyChart(
    pingMeasurements,
    "PING",
    detectTargetChanges(pingMeasurements)
);
addChartToPdf(document, pingChart, "PING Latenz über Zeit");
\end{lstlisting}
