\section{Implementierung}
\label{sec:implementierung}

\subsection{Mess-Engine und Strategy-Pattern}

Die Mess-Engine basiert auf dem \texttt{Measurer}-Interface, das alle Messtypen
vereinheitlicht (Strategy-Pattern). Jede Implementierung hat eine einzige Methode
\texttt{measure(String target)}, die ein \texttt{Measurement}-Objekt zurückgibt:

\begin{lstlisting}[language=Java, caption={Measurer-Interface}]
public interface Measurer {
    Measurement measure(String target) throws Exception;
}
\end{lstlisting}

Die drei Implementierungen:

\begin{description}
  \item[PingMeasurer] Nutzt \texttt{InetAddress.isReachable(3000)} für ICMP-Echo-Tests.
    Misst die Round-Trip-Time in Nanosekunden und konvertiert zu Millisekunden.
  \item[DnsMeasurer] Nutzt \texttt{InetAddress.getByName(hostname)} und misst die Zeit
    der DNS-Auflösung.
  \item[HttpMeasurer] Nutzt \texttt{java.net.http.HttpClient} mit konfigurierbarem
    Timeout (5\,Sekunden). Misst die Gesamtzeit des HTTP-GET-Requests.
\end{description}

\begin{lstlisting}[language=Java, caption={PingMeasurer-Implementierung}]
public class PingMeasurer implements Measurer {
    public Measurement measure(String target) throws Exception {
        long start = System.nanoTime();
        boolean reachable = InetAddress.getByName(target).isReachable(3000);
        long end = System.nanoTime();
        double latency = (end - start) / 1_000_000.0;
        return new Measurement(target, latency, reachable, "PING");
    }
}
\end{lstlisting}

\subsection{IP-Tracking und Host-Identifikation}

Jede Messung speichert zusätzlich Netzwerk-Informationen:
\begin{itemize}
  \item Lokale IPv4- und IPv6-Adressen
  \item Externe IPv4- und IPv6-Adressen (über ipify.org, mit 120-Sekunden-Cache)
  \item Eindeutiger Host-Hash (MD5 aus Hostname + OS + MAC-Adresse)
\end{itemize}

Die IP-Tracking-Logik erkennt automatisch Änderungen der externen IP und protokolliert
sie in der \texttt{ip\_changes}-Tabelle:

\begin{lstlisting}[language=Java, caption={IP-Aenderungserkennung}]
public void trackIpChange(String currentIp, String hostHash)
        throws SQLException {
    String lastIp = getLastKnownIp(hostHash);
    if (lastIp == null) {
        recordIpChange(null, currentIp, "INITIAL", hostHash);
    } else if (!lastIp.equals(currentIp)) {
        recordIpChange(lastIp, currentIp, "CHANGE", hostHash);
    }
}
\end{lstlisting}

\textbf{Hinweis zu IPv6 (DS-Lite):} Bei Providern mit DS-Lite (z.\,B.\ Magenta Telekom)
wird die externe IPv6-Adresse als ``unknown'' angezeigt, da diese Provider gemäß
RFC~8415 keine stabile IPv6-Präfixdelegation anbieten.

\subsection{Statistische Auswertung}

Die statistische Auswertung berechnet vier Kennzahlen für jeden Messtyp:

\begin{itemize}
  \item \textbf{Durchschnittliche Latenz}: $\mu = \frac{1}{n}\sum_{i=1}^{n}x_i$
  \item \textbf{95th Percentile}: Wert, unter dem 95\,\% der Messungen liegen --
    ein robusterer Indikator als der Durchschnitt, da er Ausreißer toleriert
  \item \textbf{Paketverlust}: $\frac{\text{fehlgeschlagene Messungen}}{\text{gesamte Messungen}} \times 100$
  \item \textbf{Jitter}: Durchschnitt der absoluten Differenzen zwischen
    aufeinanderfolgenden Messungen -- zeigt die Stabilität der Verbindung
\end{itemize}

\subsection{Web-Interface und Rendering-Architektur}

Das Web-Interface ist mit Javalin 5.6.3 implementiert. Um die Trennung von
Verantwortlichkeiten zu wahren, wurde das HTML-Rendering in eigenständige Klassen
ausgelagert:

\begin{description}
  \item[WebServer] (ca.\ 800 Zeilen) -- Verwaltet Javalin-Routen, REST-API-Endpunkte,
    Authentifizierungs-Middleware und Setup-Logik. Delegiert die HTML-Generierung an die
    Renderer.
  \item[HtmlPageRenderer] (ca.\ 1100 Zeilen) -- Generiert die Hauptseite mit allen
    UI-Komponenten: Live-Chart (Chart.js), Statistik-Panel, Heatmap, DNS-Benchmark,
    Konfigurations-Tabs und IP-Tracking.
  \item[SetupPageRenderer] (ca.\ 195 Zeilen) -- Generiert den Setup-Assistenten für die
    Ersteinrichtung.
\end{description}

\subsubsection{Dark Mode}

Das Theming basiert auf CSS Custom Properties (Variablen), die in zwei Blöcken
definiert sind:

\begin{lstlisting}[language=Java, caption={CSS-Variablen für Light/Dark Mode}]
:root {
    --bg-body: #f8f9fa;
    --bg-card: #ffffff;
    --color-text: #212529;
    /* ... weitere Variablen ... */
}
body.dark-mode {
    --bg-body: #1a1a2e;
    --bg-card: #16213e;
    --color-text: #e0e0e0;
    /* ... weitere Variablen ... */
}
\end{lstlisting}

Um ein \textit{Flackern} beim Laden zu vermeiden (Flash of Wrong Theme), wird die
CSS-Klasse server-seitig injiziert:

\begin{lstlisting}[language=Java, caption={Server-seitige Theme-Injection}]
boolean darkMode = Config.getInstance().getTheme().isDarkMode();
String bodyClass = darkMode ? " class=\"dark-mode\"" : "";
// ... im HTML-Template:
return """<body""" + bodyClass + """>...</body>""";
\end{lstlisting}

Chart.js-Diagramme reagieren nicht auf CSS-Variablen, daher werden deren Farben
(Achsenbeschriftung, Gitterlinien) explizit per JavaScript beim Theme-Wechsel
aktualisiert.

\subsubsection{DNS-Benchmark}

Der DNS-Benchmark vergleicht konfigurierbare DNS-Server weltweit (Google, Cloudflare,
Quad9, etc.) per paralleler Abfrage. Ein 30-Sekunden-Cooldown verhindert Missbrauch.
Die Ergebnisse werden nach Latenz sortiert und mit regionaler Farbcodierung dargestellt.

\subsection{PDF-Export}

Der PDF-Export mit OpenPDF und JFreeChart generiert professionelle Berichte mit:

\begin{itemize}
  \item Kundendaten (Name, Provider, Kundennummer)
  \item Host-Informationen (Hostname, Hash, IPs)
  \item 3 Charts (PING/DNS/HTTP) mit roten Linien bei Ziel-Änderungen
  \item Chronologische Ziel-Liste unter jedem Chart
  \item Top 10 schlechteste Messungen (sortiert nach Latenz)
  \item Verbindungsausfälle mit Gesamtanzahl
  \item Top 10 längste Verbindungsausfälle
\end{itemize}

Berichte können für drei Zeiträume generiert werden: 24 Stunden, 7 Tage und
12 Monate. Sie sind direkt als Anlage für Provider-Beschwerden verwendbar.

\subsection{Konfigurationsmanagement}

Die \texttt{Config}-Klasse ist als Singleton implementiert und verwaltet alle
Einstellungen in verschachtelten statischen Klassen:

\begin{lstlisting}[language=Java, caption={Config-Struktur (Auszug)}]
public class Config {
    private MeasurementConfig measurement;
    private MaintenanceWindow maintenanceWindow;
    private UserInfo userInfo;
    private SetupConfig setup;
    private AuthConfig auth;
    private ThemeConfig theme;  // Dark Mode
    private List<DnsServer> dnsServers;

    public static String hashPassword(String password) {
        // SHA-256 Hashing -- zentral, nicht dupliziert
    }
}
\end{lstlisting}

Die Persistenz erfolgt über Jackson JSON-Serialisierung in \texttt{config.json}.
Änderungen über die Web-UI werden sofort gespeichert.
